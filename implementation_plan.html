💡 Masukan & Saran Konstruktif (Grill Section)

⚠️ 1. Batasan Sinkronisasi Lokal (IndexedDB)

Saat ini kita menggunakan IndexedDB lokal browser yang sama untuk mensimulasikan integrasi. Di dunia nyata, LPK Admin membuka sistem dari komputer kantor LPK, sedangkan siswa membukanya dari handphone masing-masing di rumah/asrama. Di dokumen arsitektur akhir, kita harus merancang REST API terpusat (contoh: PostgreSQL/Supabase) agar sinkronisasi data lintas perangkat berjalan aman.

⚠️ 2. Keamanan Pembebasan Ranjang Asrama (Auto-Release)

Mengotomatiskan pelepasan kasur asrama 100% saat status penerbangan siswa diset "Completed" memiliki risiko tinggi jika penerbangan siswa ditunda akibat masalah visa di bandara. Ranjang siswa sebaiknya dibebaskan secara manual menggunakan tombol "Konfirmasi Penerbangan & Kosongkan Asrama" setelah status penerbangan diset selesai.

⚠️ 3. Pembuatan Akun Siswa Baru Setelah Lolos Seleksi

Ketika admin mengeklik tombol "Terima ke LPK" pada pelamar yang lolos filter, sistem sebaiknya secara otomatis men-generate Kode Aktivasi/Akun Sementara (misal dikirim via WhatsApp API) sehingga siswa tersebut bisa langsung masuk ke aplikasi belajar mereka.
